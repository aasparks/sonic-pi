Sonic Pi: WORK IN PROGRESS

John Clements <clements@racket-lang.org>
and Austin Sparks <aasparks93@gmail.com>

This package is a collection of functions and a very primitive language level that follows the lead of Sonic Pi. Specifically, (like Sonic Pi) it uses scsynth as its sound generation engine, and creates the same network of units that Sonic Pi does.

This is very much a work in progress. I’m releasing it as a project so that others can try it out and steal parts of it.

Want to try it out? Install the package, then open one of the examples in "examples/" in DrRacket and click Run.

Tell me what happens!

1 Prerequisites

For Linux users, SuperCollider and jackd must be installed on the system. Due to issues with SuperCollider, this will not work at all on a virtual machine.

2 An Imperative Interface

2.1 SuperCollider Abstraction

 (require sonic-pi/scsynth/scsynth-abstraction)
  package: sonic-pi
One thing that I don’t like that much about Sonic Pi is its unabashedly imperative interface.
However, underneath what I hope will be a nice clean language (lsonic), I’m also going to publish the lower-level imperative interface. For one thing, it’s pretty obvious what it’ll look like: it’ll look like Sonic Pi. That means less design work. Here are some functions:

procedure

(startup) → context?

Starts an scsynth, returns a context that can be used to ... well, to create a job context.

procedure

(start-job ctxt) → job-context?

  ctxt : context?
Given a context, starts a job, and return a handle for that job. A job corresponds to a “piece of music.”

procedure

(end-job job-ctxt) → void?

  job-ctxt : job-context?
Ends a job: fades out the job and cancels all scheduled notes associated with the job.

procedure

(shutdown-scsynth) → void?

Terminates the SuperCollider (and jackd on Linux) process.

procedure

(play-synth job-ctxt name time param ...) → void?

  job-ctxt : job-context?
  name : string?
  time : inexact-real?
  param : Param?
Play note or sample with job-ctxt, at time time, specified in inexact milliseconds since the epoch. If the time value is less than the current number of milliseconds since the epoch, the note will be played immediately. (Note that this means that you can always specify 0.0 to play a note immediately.)

procedure

(load-fx job-ctxt name time param ...) → void?

  job-ctxt : job-context?
  name : string?
  time : inexact-real?
  param : Param?
Create an fx node associated with job-ctxt, at time time, specified in inexact milliseconds since the epoch.

procedure

(wait-for-nodes job-ctxt) → void?

  job-ctxt : job-context?
Blocks until all synth nodes associated with job-ctxt have ended.

2.2 Data Structures

 (require sonic-pi/data-structures/common)
  package: sonic-pi

procedure

(score? scr) → boolean?

  scr : any/c
Returns true for any of the following: note, psleep, loop, sample, fx, live_loop, thread_s

 (require sonic-pi/data-structures/note)
  package: sonic-pi

procedure

(note synth-name pitch param-part ...) → note?

  synth-name : symbol?
  pitch : (or string? real?)
  param-part : (or string? real?)
Creates a note. In addition to the synth-name and pitch, users may specify non-default values for one of many other parameters using and interleaved parameter-name / value style. For instance:

(note 'saw 78 "attack" 0.5 "amp" 0.5)

The pitch may be specified as a MIDI note number, or in musical notation. Musical notation may include the note, accidental, or octave in that order :
(note 'zawa 39)
(note 'pretty_bell "A")
(note 'pulse "C2")
(note 'beep "af3")

procedure

(chord synth-name pitch interval) → (listof note?)

  synth-name : string?
  pitch : (or string? real?)
  interval : string?
creates a list of notes in the specified chord. For example:

(chord 'saw "a" "minor")

Here is a list of all supported intervals:

'("major" "minor" "major7" "dom7" "minor7" "aug" "dim" "dim7")

procedure

(control-note note param-part ...) → note?

  note : note?
  param-part : (or string? real?)
Reassigns the optional paramaters to the specified paramaters, using the note’s original values as the default.

procedure

(note? note) → boolean?

  note : any/c
returns true for notes.

2.4 Fx

 (require sonic-pi/data-structures/fx) package: sonic-pi

procedure

(fx name block param-part ...) → fx?

  name : symbol?
  block : (-> (list score?))
  param-part : (or string? real?)
Creates a sound effect to be applied to other sounds. name may be one of the predefined sound effects provided by the original Sonic Pi. Unlike with notes, all paramaters provided are passed to SuperCollider, not just the defaults. The last param-part is expected to be a block, containing a set of sounds to be played with this sound effect. For instance:

(fx 'bitcrusher
    (block (synth "beep" 60))
    "bits" 4)

procedure

(control-fx f param-part ...) → fx?

  f : fx?
  param-part : (or string? real?)
Reassigns a sound effect with new parameters. For instance:
(define my-fx (fx 'bitcrusher
                  (block (synth "beep" 60))
                  "bits" 8))
(control-fx my-fx "bits" 16)

procedure

(fx? fx) → boolean?

  fx : any/c
returns true for fx’s.

procedure

(set-fx-busses fx in-bus out-bus) → fx?

  fx : fx?
  in-bus : integer?
  out-bus : integer?
Sets the in-bus and out-bus parameters for a sound effect.

procedure

(set-block fx block) → fx?

  fx : fx?
  block : (-> (list score?))
Sets the block of music to be played for a sound effect.

3 A Live Coding Environment

This package provides a primitive language similar to Sonic Pi with a simple live coding environment within DrRacket. The provided play and stop buttons will update the playing music in real time and stop the music and shutdown SuperCollider, respectively.

3.1 Sounds

 #lang sonic-pi/lsonic package: sonic-pi

procedure

(note synth-name pitch param-part ...) → note?

  synth-name : (or symbol? string?)
  pitch : (or string? real?)
  param-part : (or string? real?)
Plays a note of the given pitch using the given synth-name. This is unlike Sonic Pi, which does not require you to input a synth each time. Users may specify non-default values for one of many other parameters using and interleaved parameter-name / value style. For instance:

(note 'saw 78 "attack" 0.5 "amp" 0.5)

The pitch may be specified as a MIDI note number, or in musical notation. Musical notation may include the note, accidental, and octave :
(note 'zawa 39)
(note 'pretty_bell "A")
(note 'pulse "C2")
(note 'beep "af3")

procedure

(chord synth-name pitch interval) → (listof note?)

  synth-name : string?
  pitch : (or string? real?)
  interval : string?
Play a chord. For example:

(chord 'saw "a" "minor")

Here is a list of all supported intervals:

'("major" "minor" "major7" "dom7" "minor7" "aug" "dim" "dim7")

procedure

(psleep time) → psleep?

  time : inexact-real?
Sleep for time seconds before playing more. The following two beeps will happen at the same time
(note 'beep 60)
(note 'beep 62)
Whereas the next two will happen one after the other, with a 1 second gap
(note 'beep 60)
(psleep 1)
(note 'beep 62)

procedure

(sample name param-part ...) → sample?

  name : (or symbol? string?)
  param-part : (or string? real?)
Play a sample. These are the same samples provided by Sonic Pi. To sample a local file, set the name to the full file path of the sample as a string. To use a built in sample, provide the name as a symbol. For instance:

(sample 'bd_tek)
(sample "/home/users/jane/documents/samples/screaming_goat.wav")

procedure

(block sounds) → (-> (listof score?))

  sounds : (-> (listof score?))
Creates a block of sound(s) similar to the do ... end blocks of Sonic Pi. This is accomplished by wrapping the set of events in a closure. Thus

(block (note 'pretty_bell 60))

is equivalent to

(λ () (list (note 'pretty_bell 60)))

blocks are used when applying fx, looping, or threading. For example:
(live_loop "foo"
           (block (sample 'bd_boom)
                  (psleep 2)))

procedure

(fx name block param-part ...) → fx?

  name : string?
  block : (-> (listof score?))
  param-part : (or string? real?)
Play block of sounds using the specified sound effect. The effects provided are the same as Sonic Pi. Each fx has a different set of parameters for it, so the defaults are determined by the synth definition. All params given will be passed and any that don’t apply to the fx will be ignored. For example:
(fx 'reverb
    (block (note 'saw "bf"))
    "room" 2)

procedure

(thread_s block) → thread_s?

  block : (-> (listof score?))
Plays the block in a separate thread from the rest of the music, allowing for simultaneous music scores.
(thread_s
 (block
  (fx 'reverb
      (block
       (loop/infinite
        (block
         (sample 'ambi_choir
                 "rate" (choose 0.5
                                (/ 1.0 3)
                                (/ 3.0 5))
                 "pan" (rrand -1 1))
         (psleep 0.5)))))))
 
(fx 'wobble
    (block
     (fx 'echo
         (block
          (loop/infinite
           (block
            (sample 'drum_heavy_kick)
            (sample 'bass_hit_c "rate" 0.8 "amp" 0.4)
            (psleep 1))))
         "mix" 0.6))
    "phase" 2)

procedure

(loop reps block) → loop?

  reps : integer?
  block : (-> (listof score?))
Loop the block reps times.

procedure

(loop/infinite block) → loop?

  block : (-> (listof score?))
Loop a block forever. Loops are blocking, so put them in a thread_s to play something else at the same time.

procedure

(control synth param-part ...) → (or note? sample? fx?)

  synth : (or note? sample? fx?)
  param-part : (or string? real?)
Set the arguments to a synth to new params and play the sound. For example, the following code will choose a note from a chord in A Minor, set the arguments, and play the note.
(control (choose (chord 'pretty_bell "a" "minor"))
         "release" 0.5
         "amp" 0.8)

procedure

(choose elem ...) → any/c

  elem : any/c
Randomly pick one element from the ones provided. If any element passed to it is a list, it will behave like choose-list. For example:
(choose (sample 'bd_haus)
        (sample 'bd_boom)
        (sample 'elec_plip))

procedure

(choose-list elements) → any/c

  elements : (listof any/c)
Same as choose but for elements specifically in a list, though choose will accept lists as well.

(choose-list (chord "beep" "a" "minor"))

procedure

(rrand min max) → real?

  min : real?
  max : real?
Generates a random number between min and max.

procedure

(rrand_i min max) → integer?

  min : integer?
  max : integer?
Generates a random integer from min to max, inclusive.

3.2 Live Coding

Live coding is implemented in a very simple fashion for use with DrRacket. There are two buttons provided with this package: stop and play.

When using the live coding environment, first press the DrRacket run button. This will start up SuperCollider and begin playing your music. If your music score is finite (no infinite loops), SuperCollider will be shut down when the music is done playing.

If there is an infinite loop, that is not a live loop, you must press the stop button to stop the music and close SuperCollider.

If there is a live loop, you can make changes to the source and press the play button to update the music on the next iteration. You must press the DrRacket run button first. It may take two iterations of the loop to update. When you wish to end the music that is playing, press the stop button. This will also shut down SuperCollider, so you will need to re-run the program to start it back up.

procedure

(live_loop name block) → live_loop?

  name : string?
  block : (-> (listof score?))
Play a block of music in an infinite loop with the ability to update. Live loops are executed in their own threads so they must have unique names and they are non-blocking. The following example comes from the Sonic Pi tutorial:

(live_loop "boom"
           (block
            (fx 'reverb
                (block (sample 'bd_boom
                               "amp" 10
                               "rate" 1)
                       (sample 'elec_blip
                               "amp" 10
                               "rate" 1)
                       (psleep 8))
                "room" 1)))
(live_loop "guit"
           (block
            (fx 'echo
                (block (sample 'guit_em9 "rate" 0.5)
                       (sample 'guit_e_fifths  "rate" 0.5)
                       (psleep 8))
                "mix" 0.3 "phase" 0.25)))
After running, comment out a sample from each loop, and press the play button to hear it change. It may take a few loops for the changes to take effect.

3.3 Examples

Shown below are the examples from the "examples" folder included with this package. Most examples come from Sonic Pi’s examples in the tutorial and on the home page. Be sure to include the lang line as

"#lang s-exp sonic-pi/lsonic"

3.3.1 synth and sleep

This block of code plays a simple set of notes in succession. The first two synths are scheduled to start at the same time. The rest will start 0.5s later with each call to psleep.
(note 'beep 60  "release" 0.5)
(note 'prophet 72 "attack" 4 "release" 2 "amp" 0.5)
(psleep 0.5)
(note 'prophet 74 "attack" 4 "release" 2 "amp" 0.5)
(psleep 0.5)
(note 'prophet 79 "attack" 4 "release" 2 "amp" 0.5)
(psleep 0.5)
(note 'prophet 77 "attack" 4 "release" 2 "amp" 0.5)
(psleep 0.5)
(note 'prophet 73 "attack" 4 "release" 2 "amp" 0.5)
(psleep 0.5)
(note 'prophet 69 "attack" 4 "release" 2 "amp" 0.5)
(psleep 0.5)
(note 'prophet 80 "attack" 4 "release" 2 "amp" 0.5)

3.3.2 sample and fx

This example plays a succession of synths and samples with various sound effects applied to them.
(synth 'pretty_bell 60)
(psleep 0.5)
(sample 'elec_plip)
(psleep 0.5)
(synth 'pretty_bell 50)
(psleep 2)
 
(fx 'reverb
    (block
     (note 'pretty_bell 60)
     (psleep 0.5)
     (sample 'elec_plip)
     (psleep 0.5)
     (note 'pretty_bell 50)))
 
(psleep 2)
 
 
(fx 'echo
    (block
     (fx 'reverb
         (block
          (note 'pretty_bell 60)
          (psleep 0.5)
          (sample 'elec_plip)
          (psleep 0.5)
          (note 'pretty_bell 50)))))
 
(psleep 4)
(note 'pretty_bell 60)
(psleep 0.5)
(sample 'elec_plip)
(psleep 0.5)
(note 'pretty_bell 50)

3.3.3 loops and threads

Loops are blocking so they must be put in a thread to be played with other loops.
(thread_s
 (block
  (loop/infinite
        (block
         (fx 'reverb
             (block
              (sample 'ambi_choir
                      "pan" (rrand -1 1)
                      "rate" (choose 0.5 (/ 1.0 3) (/ 3.0 5)))
              (psleep 0.5)))))))
(loop/infinite
      (block
       (fx 'wobble
           (block
            (fx 'echo
                (block
                 (sample 'drum_heavy_kick)
                 (sample 'bass_hit_c "rate" 0.8 "amp" 0.4)
                 (psleep 1))
                "mix" 0.6))
           "phase" 2)))

3.3.4 randomness

Randomness can be used in a number of ways to create all sorts of cool sounds. It is seeded at start up so consecutive runs should sound the same. The following example simulates ocean sounds.
(fx 'reverb
    (block
     (live_loop "ocean"
                (block
                 (note (choose 'bnoise 'cnoise 'gnoise)
                        40
                        "amp" (rrand 0.5 1.5)
                        "attack" (rrand 0 4)
                        "sustain" (rrand 0 2)
                        "release" (rrand 1 3)
                        "pan" (rrand -1 1)
                        "pan_slide" (rrand 0 1))
                 (psleep (rrand 2 3)))))
    "mix" 0.5)

3.3.5 control

Control allows you to set arguments to pre-defined synths.
(loop

 (block
  (fx 'slicer

       (fx 'reverb
           (block
            (control
             (choose (chord 'dsaw

                            "minor"))
             "release" 8
             "note_slide" 4
             "cutoff" 30
             "cutoff_slide" 4
             "detune" (rrand 0 0.2)
             "pan" (rrand -1 0))
            (control
             (choose (chord 'dsaw
                            (choose "b1" "b2" "e1" "e2" "b3" "e3")
                            "minor"))
             "release" 8
             "note_slide" 4
             "cutoff" (rrand 80 120)

4 Bugs

There are bound to be plenty of bugs, so report!

Here is a list of known bugs:

  • Loops may fall out of sync or skip an iteration due to timing delays

  • If SuperCollider (or jackd on Linux) does not have realtime privileges, timing can get messed up easily.

  • SuperCollider does not work in virtual environments. Sorry vm users!
